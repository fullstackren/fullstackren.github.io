# Memory Panel

内存面板。

应用程序的运行需要内存，因为应用程序在运行期间产生的数据需要有空间进行存储，CPU 才能与内存交互数据，保证应用程序能够运行顺畅。

## 💻 硬件层 Hardware

内存特指随机存取存储器（RAM, Random Access Memory）。一般分为**静态随机存取存储器**（SRAM, Static Random Access Memory）和**动态随机存取存储器**（DRAM, Dynamic Random Access Memory）。其中，SRAM 速度要远快于 DRAM。

在计算机的组成中主要有两种随机存取存储器：**高速缓存**（Cache）和**主存储器**（Main memory）。

- **高速缓存**（Cache）使用的就是 SRAM，但其存储容量很小（几KB~几十MB），不足以存储应用程序运行时的数据，另外由于其直接集成在 CPU 内部，这就离我们比较远，所以我们用的更多的是主存储器。
- **主存储器**（main memory，简称主存，对应我们了解的内存条）使用的是 DRAM，虽然其速度比不上高速缓存，但它存储容量大，在速度和容量上能保证同时运行几十个应用程序。

之所以要说从硬件开始说内存，是后面会涉及到这块知识。

## 🧩 软件层 Software

内存特指操作系统从主存中划出一块空间。

### 分类

按照数据类型可分为：栈内存和堆内存。

#### 栈内存 Stack Memory

栈内存的定义是因为使用了栈的结构（一种常见的数据结构，只允许在一端操作数据，遵循后进先出（Last-In-First-Out, LIFO）原则）。

- 栈内存是一段连续的内存空间，得益于栈结构简单，访问速度非常快。
- 栈内存的容量较小，主要存放函数调用信息和变量等数据，大量的内存分配操作会导致栈溢出（Stack overflow）。

#### 堆内存 Heap Memory

堆内存是一大片内存空间。

- 分配方式上，分配是动态和不连续的，数据生命周期长，
- 无用数据需要程序主动回收，否则过度占用就会造成内存泄漏（Memory Leak）。

**堆内存与栈内存的关系**

|| 堆内存 | 栈内存 |
|:-|:-|:-|
| 访问速度 | 慢 | 非常快 |
| 容量 | 大 | 小 |
| 分配 | 动态且不连续 | 连续的 |
| 存储时长 | 生命周期长 | |

**堆内存与进程关系**

一个进程只会有一个堆内存，即同一进程下的多个线程共享一个堆内存。
