# code review

## 需要评审哪些点

- 代码规范，虽然有 xxlint，但做不到全检查，比如变量命令、代码语义
- 重复的代码要抽离，做到复用
- 单个函数内容过长，需要拆分
- 算法复杂度是否可优化
- 是否有性能内存问题
- 是否有安全漏洞
- 扩展性如何
- 是否与现有的功能重复
- 是否有完善的单元测试
- 组件设计是否合理

## 评审时机

- 提交 PR 或 MR 时，通过代码 diff 进行 code review
- 每周例行一次集体 code review

## 持续优化

- 每次 code review 的问题要记录下
- 归纳整理，形成自己的代码规范体系
- 新加入的成员提前学习，提前规避
